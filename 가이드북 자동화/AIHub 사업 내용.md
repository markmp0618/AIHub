# AIHub 사업 개요

## 한 줄 정의

**AIHub**는 "AI를 뭘 써야 할지(What) + 어떻게 써서 결과를 내는지(How)"를 한 번에 연결해주는 **AI 실행 플랫폼**입니다.

단순한 툴 디렉토리가 아니라, **가이드북(학습) + 프리셋(즉시 실행) + 커뮤니티(질문/사례) + AI 추천 챗봇(개인화 내비게이션)**이 하나의 통합된 제품 경험으로 제공됩니다.

---

## 창업자 및 팀

### 창업자: 민욱
- 23살 물리학과 대학생
- AI가 직업 구조를 대체할 것이라 확신하고 창업 준비
- 강점: **학술적 분석력 + 제품/자동화 구현력(n8n, DB, 프론트엔드) + 콘텐츠 자산화 감각**
- 성향: 리스크 최소화를 선호하지만 창업에서는 과감한 도전도 필요하다고 인식

### 공동 창업자
- 석준 (역할 분담 진행 중)

---

## 해결하려는 문제

AI 툴은 매일 쏟아지지만, 대부분의 사용자는 다음과 같은 문제를 겪습니다:

1. **무엇을 써야 할지 모름** - ChatGPT, Claude, Gemini 등 선택지가 너무 많아 결정 피로 발생
2. **어떻게 써야 결과가 나오는지 모름** - 툴 소개는 많지만 실제 실행 레시피가 부족
3. **고퀄리티 워크플로우가 흩어져 있음** - 템플릿/프롬프트/자동화가 커뮤니티/깃허브/유튜브에 분산되어 검증 어려움
4. **지식이 자산으로 연결되지 않음** - 좋은 프리셋/가이드북을 만들어도 판매/공유/업데이트/피드백 루프가 단절됨

핵심은? **'탐색 → 실행 → 공유/수익화'를 한 흐름으로 연결하는 것이 문제 해결의 핵심**

## 솔루션: AIHub의 작동 방식

### 사용자 경험 플로우
1. 사용자가 "내 상황"을 입력
2. AI 추천 챗봇이 맞는 **AI 툴/가이드북/프리셋**을 묶어서 제시
3. 사용자는 **그대로 따라하면 결과물 완성**
4. 막히면 **커뮤니티에 질문**으로 해결
5. 잘 만든 사람은 프리셋을 올려 **자산화(판매/배포)**

핵심은? **AIHub는 'AI 사용법'이 아니라 'AI로 성과를 내는 루트'를 제공**

## 제품 구성 (5대 핵심 모듈)

| 모듈 | 사용자가 보는 것 | 비즈니스적 역할 |
|------|----------------|----------------|
| **AI 추천 챗봇** | 질문하면 답해주는 UI | 전환/탐색/추천을 책임지는 내비게이션 엔진 |
| **Tool Directory** | AI 도구 목록/비교 | 상단 퍼널(탐색) + SEO/콘텐츠 허브 |
| **Guidebook** | 따라하면 되는 튜토리얼 | 신뢰/브랜드/학습 루프 + 공유 유입 |
| **Preset Store** | 템플릿/워크플로우 구매 | 매출의 중심(디지털 자산 거래) |
| **Community** | 질문/사례 공유 | DAU/UGC/락인 + 신규 콘텐츠 생성 |

핵심은? **5개 모듈이 독립적이 아니라, AI 챗봇이 상황 기반으로 연결하는 것이 차별화 포인트**

## 모듈별 상세 설명

### 1. AI 추천 챗봇 (사이트의 엔진)
- 사용자 상황 입력 → DB 검색 → 관련 콘텐츠(가이드북/프리셋/도구) 추천
- 있으면 **근거 기반 추천 + 실행 플로우** 제공
- 없어도 **기본 답변**이나 **대체 경로 안내** 가능
- 구현: n8n의 AI Agent 노드 + Supabase DB 검색
- 역할: **개인화 검색/추천 레이어** (단순 대화형 UI가 아님)

#### AI 챗봇 UI 구성 (GlobalChatbot)

**레이아웃**
```
┌─────────────────────────────┐
│  [헤더]       [닫기 버튼]   │
├─────────────────────────────┤
│                             │
│  [메시지 영역]              │
│  - 초기 안내문              │
│  - 사용자 메시지 (우측)     │
│  - AI 응답 (좌측)           │
│  - 로딩 애니메이션          │
│                             │
├─────────────────────────────┤
│  [입력창]  [전송 버튼]      │
└─────────────────────────────┘
```

**기능**
- 모달 형태 UI (배경 클릭으로 닫기)
- 마크다운 렌더링: ReactMarkdown + remark-gfm
- 가이드 링크 자동 생성: `[**텍스트**]` → `/guides?search=텍스트`
- Enter 키 전송
- 자동 스크롤 (새 메시지 시)
- 로딩 애니메이션

**텍스트 정규화**
- "&" 이스케이프 처리
- 백슬래시 제거
- 굵은 텍스트를 대괄호 안으로 이동

### 2. Guidebook (가이드북)
- **목표**: 외부 툴로 가서 그대로 따라하면 완주 가능한 실행형 레시피
- **특징**: 프롬프트 팩 + 단계별 코멘트를 단계 하단에 첨부
- 핵심은? **'읽는 글'이 아니라 '실행형 레시피'로 설계**

#### 가이드북 목록 페이지 (/guides)
- **검색**: 제목, 설명, 태그, 카테고리, AI 모델명으로 검색
- **필터**: 난이도 (Lv.1 초급 / Lv.2 중급 / Lv.3 고급), 카테고리
- **프롬프트 엔지니어링 배너**: "AIHub 필수 입문" 코스 소개
- **학습 진행 추적 바**: 사용자별 진행률 표시
- **콘텐츠 형식 자동 분류**:
  - 15분 이하: 텍스트 가이드
  - 15-25분: 실습형
  - 25분 이상: 비디오 + PDF

#### 가이드북 상세 페이지 UI 구성 (/guides/:id)

**1. 상단 영역**
- Breadcrumb: 홈 > 가이드북 > 현재 가이드 제목
- 헤더 카드:
  - 가이드 제목 및 설명문
  - 진행률 (완료된 스텝 수 / 전체 스텝 수)
  - 카테고리, 관련 AI 도구 태그
  - 예상 소요시간 및 난이도

**2. 개요 섹션 (4개 카드)**
| 카드 | 내용 |
|------|------|
| 요약(Summary) | 가이드 한 줄 요약 |
| 추천 대상(Recommendations) | 이용자 유형 |
| 사전 요구사항(Requirements) | 필요한 도구/계정 |
| 핵심 원칙(Core Principles) | 핵심 학습 원칙 |

**3. 탭 전환**
- **Step-by-Step 뷰**: 단계별 학습
- **Prompt Pack 뷰**: 프롬프트 모음

**4. Step-by-Step 뷰 레이아웃**
```
┌─────────────────────────────────────────────┐
│  [좌측 사이드바]  │  [중앙 콘텐츠 영역]      │
│  - 스텝 목록      │  - 스텝 번호 및 제목    │
│  - 완료 표시(✓)   │  - 설명 (마크다운)      │
│  - 현재 스텝      │  - 목표(goal)           │
│    하이라이트     │  - 완료 기준(done_when) │
│                   │  - 팁(tips)             │
│                   │  - 체크리스트           │
│                   │  - 관련 프롬프트        │
│                   │  - 워크북 입력 필드     │
└─────────────────────────────────────────────┘
```

**5. 네비게이션 푸터**
- 이전/다음 스텝 버튼
- 완료 표시 버튼 (로그인: DB 저장 / 비로그인: localStorage)
- 진행도 텍스트

**6. 피드백 섹션 (마지막 스텝에만 표시)**
- "성공" 버튼: 축하 토스트 + 컨페티 애니메이션
- "막혔어요" 버튼: 상세 피드백 모달 오픈

**7. Prompt Pack 뷰**
- 프롬프트 카드: 텍스트, 설명, 태그, 관련 스텝 링크, 실패 패턴
- 프롬프트 클릭 시 해당 스텝으로 자동 이동

### 3. Preset Store (프리셋 스토어)
- **특징**: 다양한 프리셋 타입을 수용하는 홈 UI 설계
- 핵심은? **'콘텐츠'가 아니라 '즉시 실행 가능한 디지털 자산'**

### 4. Tool Directory (도구 디렉토리)
- Futurepedia 벤치마킹
- 가이드북/프리셋과 연결되는 관문 역할
- 핵심은? **'정보 나열'이 아니라 챗봇·가이드·프리셋으로 연결되는 허브**

#### 도구 디렉토리 페이지 기능 (/tools)

**검색 및 필터**
- 텍스트 검색: AI 도구 이름/설명 검색
- 카테고리 필터: DB에서 동적 로드
- 가격 필터: 전체 / 무료 / 유료

**정렬 옵션**
- 평점순 (별점 기준)
- 이름순 (가나다/ABC)
- 최신순 (등록일)
- 인기순 (리뷰 수)

**뷰 모드**
- 그리드 모드 (3열 레이아웃) - 기본값
- 리스트 모드 (1열 전체폭)

**도구 카드 정보**
- 로고 / 첫 글자 배지
- 도구명 및 제공사
- 평점 (별점) 및 리뷰 수
- 설명 (2줄 말줄임)
- 기능 태그 (최대 3개)
- 가격 정보

**도구 비교 기능**
- 최대 4개 도구까지 선택 가능
- 플로팅 바: 선택된 도구 썸네일 표시
- "비교하기" 버튼 → /tools/compare 페이지 이동

**대학생 추천 AI 도구 섹션**
- ChatGPT, Wrtn, Midjourney, GitHub Copilot 등 8개 도구
- 각 도구별 사용 목적 명시 (과제작성, 취업준비, 디자인 등)

### 5. Community (커뮤니티)
- **가장 중요한 행동**: 질문 올리기
- 로그인 유도도 "질문 올리기" 중심 설계
- 핵심은? **DAU를 만드는 장치이자, AIHub의 성장 엔진**

#### 커뮤니티 페이지 기능 (/community)

**게시판 분류 체계**
| 카테고리 | 설명 |
|---------|------|

## 이 폴더의 목적

이 Claude Code 프로젝트는 **가이드북과 프리셋을 자동으로 생성하는 워크플로우 자동화**를 구축하기 위한 공간입니다.

### 자동화 목표
1. 원재료(스크랩/자료)를 입력하면 구조화된 가이드북 생성
2. 프리셋(워크플로우/프롬프트)의 표준화된 템플릿 자동 생성
3. 생성된 콘텐츠를 Supabase DB에 바로 업로드 가능한 형태로 출력
4. 콘텐츠 품질 체크리스트 자동 검증

### 기대 효과
- 콘텐츠 제작 속도 10배 향상
- 품질 일관성 확보
- DB 구축 가속화
- 빠른 MVP 검증 및 피드백 루프
